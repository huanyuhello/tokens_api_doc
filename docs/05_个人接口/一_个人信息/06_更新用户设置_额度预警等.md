# 更新用户设置（额度预警等）

**PUT** `https://tokens.smartfashionai.cn/api/user/setting`

配置额度预警通知方式及相关参数。

---

## 请求参数

### Header

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `Authorization` | string | 必需 | 系统访问令牌 |
| `New-Api-User` | string | 必需 | 用户 ID（数字） |

### Body（application/json）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `notify_type` | string | 必需 | 预警类型：`email` / `webhook` / `bark` / `gotify` |
| `quota_warning_threshold` | integer | 必需 | 剩余额度低于此值时触发通知 |
| `notification_email` | string | 可选 | 通知邮箱（`email` 类型时填写） |
| `webhook_url` | string | 可选 | Webhook 地址 |
| `webhook_secret` | string | 可选 | Webhook 密钥 |
| `bark_url` | string | 可选 | Bark 推送地址 |
| `gotify_url` | string | 可选 | Gotify 服务器地址 |
| `gotify_token` | string | 可选 | Gotify 令牌 |
| `gotify_priority` | integer | 可选 | Gotify 优先级 |
| `accept_unset_model_ratio_model` | boolean | 可选 | 是否接受未设置倍率的模型 |
| `record_ip_log` | boolean | 可选 | 是否记录 IP 日志 |

---

## 请求示例

```bash
curl --location --request PUT 'https://tokens.smartfashionai.cn/api/user/setting' \
--header 'Authorization: user-token' \
--header 'New-Api-User: user-id' \
--header 'Content-Type: application/json' \
--data-raw '{
    "notify_type": "email",
    "quota_warning_threshold": 32,
    "notification_email": "email@163.com"
}'
```

---

## 响应示例

```json
{
    "data": null,
    "message": "",
    "success": true
}
```
