# API 报错代码含义

!!! note
    以下为常见 HTTP 状态码说明，具体表现可能因服务器实现略有不同。

## 报错代码一览

| 状态码 | 名称 | 含义与处理建议 |
|--------|------|----------------|
| **400** | Bad Request | 请求格式错误，通常是客户端问题。请检查请求格式，例如 `o1` 系列模型不支持 `system` 参数。 |
| **401** | Unauthorized | API 密钥验证失败，或令牌已过期。请确认 Key 是否正确；换个模型测试验证 Key 是否有效，若其他模型正常请联系管理员。 |
| **403** | Forbidden | 权限不足，令牌额度已用完或分组被禁用。编辑令牌取消额度限制，或新建令牌。 |
| **404** | Not Found | 请求的接口端点不存在。请检查 Base URL 是否正确，尝试加上 `/v1` 或末尾斜杠 `/`。 |
| **413** | Request Entity Too Large | 请求体过大，Prompt 可能超长。请缩短 Prompt 后重试。 |
| **429** | Too Many Requests | 请求频率超限。上游账号并发过高触发限流，请稍后重试。 |
| **500** | Internal Server Error | 服务器内部错误，非客户端问题。请重试，多次持续报错请联系管理员。 |
| **503** | Service Unavailable | 服务暂时不可用，可能是上游维护或该模型暂无可用渠道。请重试，持续报错请联系管理员添加渠道。 |
| **504** | Gateway Timeout | 网关超时，未能在规定时间内获得上游响应。请重试，持续报错请联系管理员。 |
| **524** | Connection Timeout | 连接超时，可能是 Tokens 通道拥挤。请重试，持续报错请联系管理员。 |

## 通用处理原则

!!! tip "遇到报错时"
    1. 先对照上表确认报错码含义
    2. 尝试重试一次
    3. 更换模型或新建令牌测试
    4. 多次持续报错请联系客服：**xiaochaozhz**
